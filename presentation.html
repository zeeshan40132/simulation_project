The Problem

Emergency rooms face constant pressure — patients arrive unpredictably, triage is complex, and doctor resources are limited. Poor resource allocation leads to long waits and worse outcomes.

We built a simulation system that lets hospital planners test different configurations (more doctors, faster triage) before implementing them in the real world.

S2

Web Worker DES Logic Zustand JavaScript
8 commits

Discrete-Event Simulation Engine

Built the full DES engine in a Web Worker — runs off the main thread so the UI never freezes even at 300× speed

Patient Lifecycle Logic

Arrival (Poisson process) → triage assignment → nurse → doctor queue → treatment → discharge or admit

Zustand State Store

Syncs worker output to React UI in real time — patients, doctors, stats and event log all update live

Live Statistics Engine

Avg wait time, doctor utilization %, queue depth, satisfaction score — all calculated continuously
